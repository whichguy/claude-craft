# Plan: Price Comparison Service

## Context

E-commerce platform needs a price comparison feature. When a user views a product,
the service queries three external supplier APIs to find the best price. Results
displayed alongside the product listing.

## Current Architecture

- Node.js API serving product pages
- PostgreSQL product catalog, prices manually updated monthly
- No supplier price integration

## Files to Change

## Implementation Steps

### Step 1: Supplier Clients

Three axios-based clients, each returning `SupplierPrice`:
- **SupplierA**: GET `https://api.supplier-a.com/v2/products/{sku}/price` with Bearer token
- **SupplierB**: POST `https://supplier-b.com/api/quote` with X-Api-Key header
- **SupplierC**: GET `https://supplierc.io/prices?sku={sku}` with Accept header

Each normalizes to `{ supplier, price, currency, inStock, shippingCost, estimatedDelivery }`.

### Step 2: Comparison Orchestrator

```typescript
export async function comparePrices(productId: string, sku: string): Promise<ComparisonResult> {
  const results = await Promise.all([
    fetchPriceFromA(sku),
    fetchPriceFromB(sku),
    fetchPriceFromC(sku)
  ]);

  const sorted = results
    .filter(r => r.inStock)
    .sort((a, b) => (a.price + a.shippingCost) - (b.price + b.shippingCost));

  return { productId, prices: results, bestPrice: sorted[0] || null, queriedAt: new Date() };
}
```

### Step 3: Route Integration

`GET /api/products/:id` fetches product from DB, then calls `comparePrices(product.id, product.sku)`.
Returns both product data and price comparison in response.

## Verification

- [ ] All three supplier clients return normalized price data
- [ ] Queries run in parallel via Promise.all
- [ ] Prices sorted by total cost (price + shipping)
- [ ] Out-of-stock items excluded from best price
- [ ] Product endpoint returns comparison alongside product
- [ ] Response time acceptable for user-facing page
