# Plan: GitHub Repository Scanner for Compliance Audit

## Context

Security team needs to scan all 1,200+ repositories in the GitHub organization for
compliance: missing branch protection, no CODEOWNERS, outdated CI configs, exposed
secrets in public repos.

## Current Architecture

- Manual spot-checks by security team
- GitHub Enterprise with PAT-based API access
- Results tracked in a Google Sheet (manually updated)

## Files to Change

## Implementation Steps

### Step 1: GitHub Client

Paginated repo listing via Octokit (`per_page: 100`, loop until < 100 results).
Helper functions for `getFileContent`, `getBranchProtection`.

### Step 2: Compliance Checks

Each check returns `{ check, repo, pass, issues[] }`:
- **Branch protection**: PR reviews required? Status checks? Admin enforcement?
- **CODEOWNERS**: Check root and `.github/` locations
- **CI config**: `.github/workflows` directory exists?
- **Public repo secrets**: Pattern match on file contents

### Step 3: Scanning Orchestrator

```typescript
export async function scanAllRepos(org: string): Promise<ScanReport> {
  const repos = await githubClient.getAllRepos(org);
  const results: CheckResult[] = [];

  for (const repo of repos) {
    results.push(await checkBranchProtection(repo));
    results.push(await checkCodeowners(repo));
    results.push(await checkCiConfig(repo));
    if (repo.visibility === 'public') {
      results.push(await checkPublicRepoSecrets(repo));
    }
  }

  return generateReport(results);
}
```

Processes repos sequentially, running 3-4 API calls per repo.

### Step 4: Report Generator

Groups results by repo, summarizes pass/fail counts, writes CSV with
`repo,check,status,issues` columns.
