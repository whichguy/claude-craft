# Plan: Restructure Application Configuration Schema

## Context

Our application configuration has grown organically to 45+ flat keys in a single
`config.json` file. Keys like `db_host`, `db_port`, `db_name`, `redis_host`, `redis_port`,
`smtp_host`, etc. are hard to manage. We need to restructure into nested groups for
maintainability and to support environment-specific overrides.

## Current State

- Node.js 20, TypeScript 5.3
- Config loaded from `config.json` at startup via `src/config/loader.ts`
- Flat structure: `{ "db_host": "localhost", "db_port": 5432, "db_name": "app", ... }`
- 45 keys across database, cache, email, auth, and feature flag categories
- `ConfigLoader.get(key)` used in ~30 files across the codebase
- 12 production deployments using current flat config format
- Config files stored in `/etc/app/config.json` on each server

## Approach

Restructure the configuration schema to use nested objects grouped by concern. Update
the config loader to support dot-notation access for the nested structure. Add TypeScript
types for compile-time safety.

## Files to Modify

- `src/config/schema.ts` (new) — TypeScript interfaces for nested config
- `src/config/loader.ts` — rewrite to parse nested config, support dot-notation access
- `src/config/validator.ts` (new) — JSON Schema validation for new format
- `src/config/defaults.ts` (new) — default values for all config keys
- `config.json` — restructure from flat to nested format
- `config.test.json` — test environment config in new format
- `src/types/config.ts` — export typed config accessors

## Implementation

### Phase 1: New Schema Definition

1. Define nested config schema in `src/config/schema.ts`:
   ```typescript
   interface AppConfig {
     database: {
       host: string;
       port: number;
       name: string;
       pool: { min: number; max: number; idleTimeout: number };
     };
     cache: {
       host: string;
       port: number;
       ttl: number;
       prefix: string;
     };
     email: {
       host: string;
       port: number;
       from: string;
       auth: { user: string; pass: string };
     };
     auth: {
       jwtSecret: string;
       tokenExpiry: number;
       refreshExpiry: number;
     };
     features: Record<string, boolean>;
   }
   ```

2. Create `defaults.ts` with sensible defaults for development:
   - Database: localhost:5432/app_dev
   - Cache: localhost:6379, 300s TTL
   - Email: localhost:1025 (MailHog)

3. Create `validator.ts` using `ajv` to validate config against schema:
   - Required fields check
   - Type validation (port is number, host is string)
   - Range validation (port 1-65535, TTL > 0)

### Phase 2: Loader Rewrite

1. Rewrite `loader.ts` to parse nested JSON:
   - Load `config.json`, merge with defaults (deep merge)
   - Support environment variable overrides: `APP_DATABASE__HOST` → `database.host`
   - Support dot-notation access: `config.get('database.host')`
   - Typed accessor: `config.getTyped<DatabaseConfig>('database')`

2. Add config freezing after load: `Object.freeze(deepFreeze(config))`

3. Add `config.getSection(name)` for getting an entire group

### Phase 3: Update Config Files

1. Restructure `config.json` to nested format:
   ```json
   {
     "database": { "host": "prod-db.internal", "port": 5432, "name": "app_prod" },
     "cache": { "host": "prod-redis.internal", "port": 6379, "ttl": 600 }
   }
   ```

2. Create matching `config.test.json` for test environment

3. Update all 30 call sites from `config.get('db_host')` to `config.get('database.host')`

### Phase 4: Cleanup

1. Remove old flat-key parsing logic from loader
2. Update environment variable documentation
3. Add config validation to application startup (fail-fast on invalid config)

## Verification

1. Unit test loader: nested config parsing, dot-notation access, deep merge
2. Unit test validator: valid config passes, missing fields rejected
3. Unit test environment variable override mapping
4. Integration test: full application startup with new config format
5. Verify typed accessors return correct TypeScript types
6. Test config freezing: mutation attempts throw errors
