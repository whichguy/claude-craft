# Plan: Refactor formatCurrency() Signature

## Context

`formatCurrency(amount)` currently formats to USD only. We need to support
multi-currency formatting. Changing the signature to `formatCurrency(amount, currency)`.

## Current State

- `src/utils/currency.ts` — exports `formatCurrency(amount: number): string`
- Called by 5 consumers:
  - `src/components/PriceDisplay.tsx` — renders product prices
  - `src/components/CartTotal.tsx` — renders cart summary
  - `src/services/invoice-service.ts` — generates PDF invoices
  - `src/services/email-templates.ts` — formats amounts in emails
  - `src/api/routes/orders.ts` — formats amounts in API responses

## Approach

Add optional `currency` parameter with 'USD' default for backward compatibility.
Update all 5 callers to pass explicit currency from their data context.

## Files to Modify

- `src/utils/currency.ts` — update signature
- `src/components/PriceDisplay.tsx` — pass product.currency
- `src/components/CartTotal.tsx` — pass cart.currency
- `src/services/invoice-service.ts` — pass order.currency
- `src/services/email-templates.ts` — pass order.currency
- `src/api/routes/orders.ts` — pass order.currency

## Implementation

1. Update `formatCurrency` signature: add `currency: string = 'USD'`
2. Implement Intl.NumberFormat-based formatting with currency parameter
3. Update all 5 callers to pass explicit currency
4. Add new currencies to supported list: EUR, GBP, JPY, CAD

## Verification
