# Form 990 Skill

Multi-phase orchestrator skill (`/form990`) for preparing IRS Form 990 nonprofit tax returns. Manages P0–P9 phase pipeline via a plan-file state machine, drives Google Sheets/Drive/Gmail MCP tools, produces structured JSON datasets for e-file providers, and applies a 3-tier CPA quality gate (Q-F1..Q-F18). Designed as a resumable, crash-safe workflow with PII-aware artifact handling.

- **From Session d063875e:** Built the full skill: SKILL.md orchestrator (subcommands: init/resume/phase/review/status/ask), PHASES.md (P0–P9 playbooks), QUESTIONS.md (CPA gate catalog Q-F1..Q-F18 in Gate-1/Gate-2/Gate-3 tiers), SCHEDULES.md (Schedule A/B/G/O playbooks), LEARNINGS.md, TOOL-SIGNATURES.md, PERSONA.md, plus templates (PLAN-TEMPLATE.md, efile-handoff-packet.md, email-question.md). Key architectural decisions: (a) the "PDF is reference only — MeF is the filing channel" pivot must appear as a prominent P0 banner because users uniformly expect to file via PDF; (b) plan-file machine state uses Content-SHA256 CAS (not flock) because macOS flock is advisory and any editor can overwrite mid-session; (c) the revalidation circuit breaker is fact-centric — 3 consecutive regressions of the same fact_id halt with AskUserQuestion regardless of whether the regression was detected at resume time or inside a phase Pre-check; (d) data-heavy phases (P2 chart-of-accounts mapping, P3 financials, P6 Schedule A public-support test) use the Programmatic Analysis pattern (Python stdlib scripts) instead of inline LLM arithmetic.
