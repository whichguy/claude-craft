# Plan: Compiled Template Cache for Rendering Engine

## Context

Node.js app uses Handlebars for SSR. Template compilation takes 15-40ms per template
across 85 templates. Cold start is ~2s. Cache compiled templates to disk for fast
subsequent starts.

## Current Architecture

- Express.js with `express-handlebars` view engine
- 85 `.hbs` template files in `views/`
- Templates compiled on first use (lazy), no caching

## Files to Change

| File | Change |
|------|--------|
| `src/templateEngine.ts` | Compiled template cache manager |
| `src/middleware/renderMiddleware.ts` | Use cached templates |
| `scripts/precompile.ts` | Build script to warm cache |
| `cache/templates/` | Compiled output directory |
| `package.json` | Add `precompile` script |

## Implementation Steps

### Step 1: Template Cache Manager

```typescript
const CACHE_DIR = 'cache/templates';

export function getCompiledTemplate(name: string): HandlebarsTemplateDelegate {
  const cachePath = path.join(CACHE_DIR, `${name}.compiled.js`);

  if (fs.existsSync(cachePath)) {
    const spec = require(cachePath);
    return Handlebars.template(spec);
  }

  // Fallback: compile from source and write to disk cache
  const source = fs.readFileSync(path.join('views', `${name}.hbs`), 'utf8');
  const compiled = Handlebars.precompile(source);
  fs.mkdirSync(CACHE_DIR, { recursive: true });
  fs.writeFileSync(cachePath, `module.exports = ${compiled};`);
  return Handlebars.compile(source);
}
```

### Step 2: Render Middleware

Replace direct `Handlebars.compile()` calls with `getCompiledTemplate()`.

### Step 3: Precompile Build Script

Glob all `views/**/*.hbs`, precompile each, write to `cache/templates/`.
Added to build pipeline: `"build": "tsc && npm run precompile"`.

### Step 4: Gitignore and CI

`cache/templates/` in `.gitignore`. Precompile runs in CI after `tsc`.
