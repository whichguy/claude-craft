# Plan: Widget Library — Create & Render Pipeline

## Context

We are building a widget library that lets users create UI widgets (buttons, cards,
charts) via a programmatic API and render them into a dashboard. The library will be
used by three internal teams. Phase 1 creates the widget factory, Phase 2 builds the
rendering engine that consumes widgets from Phase 1.

## Current State

- TypeScript 5.3, React 18 for rendering
- No existing widget system — greenfield library
- Dashboard currently uses hardcoded React components
- Target: replace hardcoded components with configurable widgets
- Three consuming teams have agreed on the API design

## Approach

Two-phase build: first implement the widget creation API that produces widget
descriptors, then build the renderer that takes descriptors and produces React
components. The creation API will be pure TypeScript (no React dependency) so it
can be used in Node.js backend contexts too.

## Files to Modify

- `src/widgets/factory.ts` (new) — widget creation functions
- `src/widgets/types.ts` (new) — widget descriptor types
- `src/widgets/registry.ts` (new) — widget type registry
- `src/widgets/renderer.ts` (new) — React rendering engine
- `src/widgets/components/Button.tsx` (new) — button widget renderer
- `src/widgets/components/Card.tsx` (new) — card widget renderer
- `src/widgets/components/Chart.tsx` (new) — chart widget renderer
- `src/dashboard/WidgetGrid.tsx` (new) — dashboard layout using widgets

## Implementation

### Phase 1: Widget Creation API

1. Define widget types in `src/widgets/types.ts`:
   ```typescript
   interface WidgetDescriptor {
     id: string;
     type: string;
     props: Record<string, unknown>;
     layout: { row: number; col: number; width: number; height: number };
   }
   ```

2. Create `factory.ts` with widget creation functions:
   - `createWidget(name: string, type: string): WidgetDescriptor`
   - Takes positional arguments: name (display label) and type (widget kind)
   - Generates unique ID via `crypto.randomUUID()`
   - Returns a WidgetDescriptor with default layout and empty props

3. Create `registry.ts` for widget type registration:
   - `registerType(type: string, defaultProps: Record<string, unknown>)`
   - `getRegisteredTypes(): string[]`
   - Built-in types: `button`, `card`, `chart`

4. Add factory helpers:
   - `createButton(name)` — shorthand for `createWidget(name, 'button')`
   - `createCard(name)` — shorthand for `createWidget(name, 'card')`
   - `createChart(name)` — shorthand for `createWidget(name, 'chart')`

### Phase 2: Rendering Engine

1. Create `renderer.ts` — the main rendering orchestrator:
   - `renderWidget(descriptor: WidgetDescriptor): React.ReactElement`
   - Looks up component by widget type in component registry
   - Passes descriptor props to the matched component
   - Widget creation call: `createWidget({ name, type, color })` for full control
   - Handles unknown types with a fallback "Unknown Widget" component

2. Build individual widget components:
   - `Button.tsx`: renders a styled button with configurable label, color, onClick
   - `Card.tsx`: renders a card with title, body, footer sections
   - `Chart.tsx`: renders a chart (bar/line/pie) using recharts library

3. Create `WidgetGrid.tsx` for dashboard layout:
   - Takes `WidgetDescriptor[]` and renders them in a CSS Grid
   - Respects layout properties (row, col, width, height)
   - Supports drag-and-drop reordering (react-dnd)
   - Responsive breakpoints for mobile/tablet/desktop

### Phase 3: Dashboard Integration

1. Replace hardcoded dashboard components with widget descriptors
2. Load widget configurations from user preferences API
3. Add "Add Widget" modal with type selection and configuration form
4. Persist widget layout changes to backend

## Verification

1. Unit test factory: `createWidget` returns valid descriptor with correct fields
2. Unit test registry: register type, retrieve defaults
3. Unit test renderer: each widget type renders correct component
4. Unit test fallback: unknown type renders "Unknown Widget" placeholder
5. Integration test: create widget → render → verify DOM output
6. Visual regression test: screenshot comparison for each widget type
