# Plan: Real-Time Dashboard with WebSocket Streaming

## Context

Our analytics dashboard currently polls the API every 30 seconds for updated metrics.
Users report stale data and want real-time updates. We need to add WebSocket streaming
for live metric updates.

## Current State

- React frontend with polling-based data fetching
- Express API serving metric aggregations from ClickHouse
- No existing WebSocket infrastructure

## Approach

Use Socket.IO for WebSocket connections. ClickHouse supports streaming queries via its
HTTP interface, so we'll pipe ClickHouse query results directly to connected clients
through Socket.IO rooms. Redis pub/sub will handle multi-instance fan-out.

## Files to Modify

- `src/server/websocket.ts` (new) — Socket.IO setup and room management
- `src/server/metric-stream.ts` (new) — ClickHouse streaming query wrapper
- `src/client/hooks/useMetricStream.ts` (new) — React hook for WebSocket data
- `src/client/components/Dashboard.tsx` — replace polling with WebSocket
- `package.json` — add socket.io, socket.io-client, ioredis

## Implementation

### Phase 1: WebSocket Infrastructure

1. Install dependencies: `npm install socket.io socket.io-client ioredis`
2. Create Socket.IO server attached to Express:
   ```ts
   const io = new Server(httpServer, {
     cors: { origin: process.env.DASHBOARD_URL }
   });
   ```
3. Set up Redis adapter for multi-instance support:
   ```ts
   import { createAdapter } from '@socket.io/redis-adapter';
   const pubClient = new Redis(process.env.REDIS_URL);
   const subClient = pubClient.duplicate();
   io.adapter(createAdapter(pubClient, subClient));
   ```
4. The Socket.IO Redis adapter handles reconnection automatically, so we don't
   need explicit error handling for Redis connection drops.

### Phase 2: ClickHouse Streaming

1. Create streaming query wrapper using ClickHouse HTTP chunked transfer:
   ```ts
   async function streamMetrics(query: string, socket: Socket) {
     const response = await fetch(CLICKHOUSE_URL, {
       method: 'POST',
       body: query,
       headers: { 'Transfer-Encoding': 'chunked' }
     });
     // ClickHouse supports chunked streaming responses
     const reader = response.body.getReader();
     while (true) {
       const { done, value } = await reader.read();
       if (done) break;
       socket.emit('metric-update', JSON.parse(new TextDecoder().decode(value)));
     }
   }
   ```
2. ClickHouse will handle backpressure natively through HTTP/2 flow control.

### Phase 3: Client Integration

1. Create `useMetricStream` hook:
   ```ts
   function useMetricStream(metricId: string) {
     const socket = useSocket();
     const [data, setData] = useState(null);
     useEffect(() => {
       socket.emit('subscribe', metricId);
       socket.on('metric-update', setData);
     }, [metricId]);
     return data;
   }
   ```
2. Replace `useQuery` polling in Dashboard.tsx with `useMetricStream`
3. Performance should be fine for up to 500 concurrent WebSocket connections
   per server instance.
