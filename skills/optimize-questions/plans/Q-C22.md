# Plan: Google Drive File Export for GAS Inventory System

## Context

The inventory tracking GAS project needs "Export to Drive". Users click a sidebar
button to export the current report as a PDF to Google Drive. Requires Drive API
access from the existing Sheets-bound script. ~30 active users.

## Current Architecture

- Sheets-bound GAS project (ID: 1DEF...UVW)
- 6 existing .gs files for inventory CRUD
- OAuth scopes: `spreadsheets` only
- Sidebar UI built with HtmlService

## Files to Change

| File | Change |
|------|--------|
| `src/export.gs` | New Drive export functions |
| `src/sidebar.html` | Add "Export to PDF" button |
| `src/sidebar.js` | Client-side export handler |
| `appsscript.json` | Add Drive API scope |

## Implementation Steps

### Step 1: Update Manifest

Add `https://www.googleapis.com/auth/drive` scope to `appsscript.json` alongside
existing `spreadsheets` scope.

### Step 2: Export Module

`exportReportToPdf()`:
- Generate PDF blob via Sheets export URL with OAuth token
- Save to designated Drive folder (auto-created on first use)
- Filename: `Inventory-Report-YYYY-MM-DD.pdf`
- Folder ID stored in ScriptProperties for reuse

### Step 3: Sidebar Button

New "Export Report as PDF" button. Click handler uses `google.script.run` with
success/failure callbacks. Shows status: "Exporting..." then filename or error.

### Step 4: Folder Management

Private helper `getExportFolderId_()`:
- Check ScriptProperties for stored folder ID
- If missing, create "Inventory Exports" folder via DriveApp
- Store ID for future use
