# Plan: Data Aggregation Service — Multi-Source Dashboard

## Context

Executive dashboard pulls data from three external APIs: Salesforce (revenue), Jira
(velocity), and Google Analytics (traffic). Fetched hourly, stored for historical
display.

## Current Architecture

- Next.js dashboard with PostgreSQL for historical metrics
- Manual data entry via admin panel
- No external API integrations

## Implementation Steps

### Step 1: Metrics Table

`metrics`: source, metric_name, metric_value, recorded_at, raw_payload (JSONB).
Index on (source, recorded_at).

### Step 2: Salesforce Client

OAuth token exchange, then SOQL query for monthly opportunity sum via REST API.

### Step 3: Jira Client

GET sprint report via Agile REST API with basic auth. Extract completed vs total points.

### Step 4: Google Analytics Client

Use `@google-analytics/data` BetaAnalyticsDataClient, run report for sessions and
active users over date range.

### Step 5: Transformer

Normalize each source's response into `{ source, metricName, metricValue, timestamp }` records.

### Step 6: Scheduler

```typescript
cron.schedule('0 * * * *', async () => {
  const revenue = await fetchRevenue('THIS_MONTH');
  const velocity = await fetchVelocity(getCurrentSprintId());
  const traffic = await fetchTraffic('7daysAgo');

  const metrics = [
    ...transformToMetrics(revenue),
    ...transformToMetrics(velocity),
    ...transformToMetrics(traffic)
  ];
  await saveMetrics(metrics);
});
```
