# Plan: Log Aggregator with Background Workers

## Context

Our distributed system produces logs across 12 microservices. Currently each service
writes to its own log file. We need a centralized log aggregator that collects,
merges, and rotates logs into a single daily file for analysis.

## Current Architecture

- 12 Node.js microservices, each writing to `/var/log/{service}.log`
- No centralized log collection
- Manual log review via SSH to each host
- Logs lost on container restart

## Files to Change

| File | Change |
|------|--------|
| `src/workers/logCollector.ts` | New background worker to poll service logs |
| `src/workers/logWriter.ts` | New background worker to flush buffer to daily file |
| `src/aggregator.ts` | Main orchestrator: starts workers, manages shared buffer |
| `src/utils/logParser.ts` | Parse heterogeneous log formats |
| `config/services.json` | Service endpoints and log paths |
| `output/daily/{date}.log` | Aggregated output file |

## Approach

Two background workers run on independent intervals:
1. **Collector** — polls each service every 5 seconds, appends new lines to a shared
   in-memory buffer, and tracks read offsets per service.
2. **Writer** — every 10 seconds, drains the buffer and appends to the daily output
   file at `output/daily/YYYY-MM-DD.log`.

Both workers operate on the same output file.

## Implementation Steps

### Step 1: Shared Buffer

Create an in-memory array in `aggregator.ts`:
```typescript
const logBuffer: string[] = [];
```
Both workers import this shared reference.

### Step 2: Log Collector Worker

`src/workers/logCollector.ts`:
- Runs via `setInterval(collectLogs, 5000)`
- For each service in `config/services.json`:
  - Read log file from last known offset
  - Parse each line with `logParser.parse(line)`
  - Push formatted entries to `logBuffer`
  - Update offset tracker
- Handles missing files gracefully (service may not have logged yet)

### Step 3: Log Writer Worker

`src/workers/logWriter.ts`:
- Runs via `setInterval(writeLogs, 10000)`
- Drains `logBuffer` (splice all entries)
- Opens daily file with `fs.appendFileSync(dailyPath, entries.join('\n'))`
- Logs write count to stdout

### Step 4: Daily File Rotation

The writer determines the filename from current date:
```typescript
const dailyPath = `output/daily/${format(new Date(), 'yyyy-MM-dd')}.log`;
```
At midnight, the path naturally rotates to a new file.

### Step 5: Orchestrator

`src/aggregator.ts`:
- Validates config on startup
- Creates output directory if missing
- Starts both workers
- Handles SIGTERM: flushes remaining buffer, then exits

### Step 6: Log Parser

`src/utils/logParser.ts`:
- Normalizes timestamps to ISO 8601
- Extracts level (INFO/WARN/ERROR)
- Prefixes with service name
- Output format: `[ISO_TIMESTAMP] [SERVICE] [LEVEL] message`

## Configuration

```json
{
  "services": [
    { "name": "auth-service", "logPath": "/var/log/auth-service.log" },
    { "name": "order-service", "logPath": "/var/log/order-service.log" }
  ],
  "collectIntervalMs": 5000,
  "writeIntervalMs": 10000,
  "outputDir": "output/daily"
}
```

## Verification

- [ ] Collector reads new log entries from each service
- [ ] Writer flushes buffer to daily file
- [ ] Daily rotation produces new file at midnight
- [ ] SIGTERM gracefully flushes remaining buffer
- [ ] Log format is consistent and parseable
- [ ] Handles missing/empty service log files

## Rollback

- Stop aggregator process
- Services continue writing to their own logs (no impact)
- Remove output directory if needed
