# Plan: Data Analysis CLI Tool with Query DSL

## Context

Our data team currently writes ad-hoc Python scripts to query our analytics warehouse. We need a CLI tool that provides a structured query DSL for common analysis patterns, wrapped in a user-friendly command-line interface. The DSL is the core value — it lets analysts express complex queries (joins, aggregations, window functions) in a concise, composable syntax.

## Approach

Build a TypeScript CLI tool with two layers: (1) a query DSL that compiles to SQL, and (2) a CLI that parses user arguments and invokes the DSL. The DSL supports chaining, aggregation groups, and subquery composition.

## Files to Modify

- `src/dsl/query.ts` — query DSL engine (new)
- `src/cli/parser.ts` — argument parsing and validation (new)
- `src/cli/commands/` — individual command handlers (new)
- `src/cli/formatters/` — output formatting modules (new)
- `src/db/warehouse.ts` — warehouse connection adapter (new)
- `package.json` — dependencies and bin entry
- `tsconfig.json` — compiler configuration

## Implementation

### Phase 1: Query DSL Core

1. Create `src/dsl/query.ts` with the chainable query builder that compiles to SQL.
   Support `select`, `from`, `where`, `groupBy`, `orderBy`, `limit`, `join`, and
   `window` methods.

### Phase 2: CLI Argument Parsing

2. Install CLI dependencies:
   ```
   npm install commander chalk ora inquirer conf cli-table3
   npm install @types/inquirer @types/chalk
   ```

3. Create `src/cli/parser.ts` — the main CLI entry point:
   - Use `commander` to define the program structure
   - Set program name to `dqa` (data query analyzer)
   - Set version from `package.json`
   - Configure global options that apply to all commands:
     ```typescript
     program
       .option('-o, --output <format>', 'Output format', 'table')
       .option('-c, --connection <name>', 'Named connection profile', 'default')
       .option('--dry-run', 'Print SQL without executing', false)
       .option('--no-color', 'Disable colored output')
       .option('--verbose', 'Enable verbose logging', false)
       .option('--timeout <ms>', 'Query timeout in milliseconds', '30000')
       .option('--max-rows <n>', 'Maximum rows to return', '1000')
       .option('--profile <name>', 'Configuration profile to use', 'default')
     ```

4. Implement connection profile management in `src/cli/commands/config.ts`:
   - `dqa config add <name>` — add a named connection profile interactively
   - `dqa config list` — show all saved profiles
   - `dqa config remove <name>` — delete a profile
   - `dqa config test <name>` — test connectivity
   - Store profiles using `conf` in `~/.dqa/profiles.json`
   - Support environment variable interpolation in profile values
   - Validate connection strings before saving

5. Implement the `query` command in `src/cli/commands/query.ts`:
   - `dqa query <expression>` — parse DSL expression string from argument
   - `dqa query -f <file>` — read DSL expression from a `.dqa` file
   - `dqa query --interactive` — enter interactive REPL mode
   - Support piping: `echo "users | where age > 30" | dqa query`
   - Add `--explain` flag to show query execution plan
   - Add `--cache` flag to cache results for 5 minutes (configurable)
   - Add `--bookmark <name>` to save the query for reuse

6. Implement the `explore` command in `src/cli/commands/explore.ts`:
   - `dqa explore <table>` — show table schema and sample data
   - `dqa explore <table> --stats` — show column statistics (min, max, avg, null%)
   - `dqa explore --search <pattern>` — find tables matching a pattern
   - `dqa explore <table> --relationships` — show foreign key relationships
   - Display results using `cli-table3` with colored output

7. Implement the `history` command in `src/cli/commands/history.ts`:
   - `dqa history` — show recent queries with execution time
   - `dqa history --search <term>` — filter history by keyword
   - `dqa history --replay <id>` — re-execute a previous query
   - `dqa history --export` — export history to CSV
   - Store history in `~/.dqa/history.json` with timestamps

8. Implement the `schedule` command in `src/cli/commands/schedule.ts`:
   - `dqa schedule add <cron> <query>` — schedule recurring query execution
   - `dqa schedule list` — show active schedules
   - `dqa schedule remove <id>` — cancel a schedule
   - `dqa schedule logs <id>` — view execution logs for a schedule
   - Uses node-cron for scheduling, results emailed or written to file

### Phase 3: Output Formatting

9. Create output formatter interface in `src/cli/formatters/base.ts`:
   ```typescript
   interface OutputFormatter {
     format(rows: Row[], columns: Column[]): string;
     supportsStreaming: boolean;
     fileExtension: string;
   }
   ```

10. Implement `src/cli/formatters/table.ts`:
    - ASCII table output using `cli-table3`
    - Automatic column width detection based on terminal width
    - Truncate long values with `...` indicator
    - Color-code null values in yellow, numbers in cyan
    - Support for horizontal and vertical table orientations
    - Page output if rows exceed terminal height

11. Implement `src/cli/formatters/csv.ts`:
    - Standard CSV output with configurable delimiter
    - Proper escaping of commas, quotes, and newlines
    - Optional header row (enabled by default)
    - Support streaming output for large result sets
    - BOM option for Excel compatibility

12. Implement `src/cli/formatters/json.ts`:
    - Pretty-printed JSON output (default for `--output json`)
    - JSON Lines format for streaming (`--output jsonl`)
    - Configurable indentation level
    - Optional JSON Schema generation from results

13. Implement `src/cli/formatters/markdown.ts`:
    - GitHub-flavored markdown table output
    - Useful for pasting query results into PRs and issues
    - Support column alignment (left, right, center) based on data type

14. Create formatter factory in `src/cli/formatters/index.ts`:
    - Registry pattern for formatters
    - Auto-detect best format when output is piped vs terminal
    - Allow custom formatters via plugin system

### Phase 4: CLI Shell and Interactive Features

15. Implement interactive REPL in `src/cli/repl.ts`:
    - Tab completion for table names, column names, and DSL keywords
    - Syntax highlighting for DSL expressions using `chalk`
    - Multi-line input support with `\` continuation
    - `.help`, `.tables`, `.schema <table>`, `.quit` meta-commands
    - Persistent history across sessions
    - Auto-suggest based on recent queries

16. Add progress indicators in `src/cli/progress.ts`:
    - Spinner using `ora` while queries execute
    - Row count progress for long-running queries
    - ETA estimation based on query plan

17. Implement `src/cli/autocomplete.ts`:
    - Generate shell completion scripts for bash, zsh, and fish
    - `dqa completion install` — auto-install completions
    - Context-aware completions (tables after `from`, columns after `where`)

### Phase 5: Warehouse Connection

18. Create `src/db/warehouse.ts`:
    - Connection pool management
    - Support PostgreSQL, MySQL, and BigQuery adapters
    - Query execution with timeout and cancellation support
    - Result streaming for large datasets

## Verification

- Run `npm test` — all unit tests pass
- `dqa query "users | where signup > '2024-01-01' | count"` returns correct result
- `dqa explore users --stats` shows column statistics
- `dqa query --dry-run` prints SQL without executing
- Output formats produce valid CSV, JSON, and markdown
- Interactive REPL tab-completes table and column names
