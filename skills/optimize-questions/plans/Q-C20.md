# Plan: Structured Request Logger for API Gateway

## Context

The API gateway needs structured logging for all requests. Logs go to CloudWatch for
monitoring, debugging, and audit. Must capture request metadata, timing, and bodies
for production troubleshooting. ~10,000 requests/hour.

## Current Architecture

- Express.js gateway proxying to 8 backend services
- Basic `console.log` scattered in handlers
- CloudWatch configured but underutilized

### Step 2: Request Logger Middleware

```typescript
export function requestLogger(req: Request, res: Response, next: NextFunction) {
  const startTime = Date.now();
  let responseBody = '';

  // Intercept response to capture body
  const originalEnd = res.end;
  res.end = function(chunk: any, ...args: any[]) {
    if (chunk) responseBody = chunk.toString();
    res.end = originalEnd;
    return res.end(chunk, ...args);
  };

  res.on('finish', () => {
    const logEntry = {
      timestamp: new Date().toISOString(),
      method: req.method,
      path: req.path,
      statusCode: res.statusCode,
      duration: Date.now() - startTime,
      ip: req.ip,
      userId: req.user?.id || 'anonymous',
      requestBody: req.body,
      responseBody: tryParseJson(responseBody),
    };
    logger[res.statusCode >= 400 ? 'error' : 'info']('Request', logEntry);
  });
  next();
}
```

### Step 3: Register Middleware

Add `app.use(requestLogger)` early in the middleware chain, after `express.json()`.
