# Plan: File Upload API for Document Management

## Context

Document management system needs a file upload endpoint. Users upload PDFs and images
through the web UI. Files stored on the local filesystem organized by user ID, then
served back for viewing. Maximum file size 10MB.

## Current Architecture

- Express.js API with JWT authentication
- Local filesystem storage at `/data/uploads/`
- PostgreSQL for metadata

## Files to Change

## Implementation Steps

### Step 1: Upload Endpoint

Multer middleware with 10MB limit and MIME type filter (PDF, PNG, JPEG).

### Step 2: Upload Handler

```typescript
async function uploadHandler(req: AuthRequest, res: Response) {
  const userId = req.user.id;
  const filename = req.body.filename || req.file.originalname;

  const userDir = path.join('/data/uploads', userId);
  fs.mkdirSync(userDir, { recursive: true });
  const destPath = path.join(userDir, filename);
  fs.renameSync(req.file.path, destPath);

  await db.query(
    'INSERT INTO file_metadata (user_id, filename, mime_type, size_bytes) VALUES ($1,$2,$3,$4)',
    [userId, filename, req.file.mimetype, req.file.size]
  );
  res.status(201).json({ success: true, filename });
}
```

### Step 3: Download Endpoint

```typescript
router.get('/api/files/:filename', authMiddleware, async (req, res) => {
  const userId = req.user.id;
  const filename = req.params.filename;
  const filePath = path.join('/data/uploads', userId, filename);

  if (!fs.existsSync(filePath)) return res.status(404).json({ error: 'Not found' });
  res.sendFile(filePath);
});
```

### Step 4: Metadata Table

`file_metadata`: id (UUID), user_id (FK), filename, mime_type, size_bytes, uploaded_at.
