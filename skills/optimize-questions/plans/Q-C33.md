# Plan: Webhook Forwarding Service

## Context

Lightweight service receiving webhooks from GitHub, Stripe, and Sentry, normalizing
payloads, and forwarding to internal Slack channels. Each provider maps to a specific
channel.

## Current Architecture

- No webhook processing service
- Developers manually check dashboards
- Internal services run on Kubernetes

## Files to Change

| File | Change |
|------|--------|
| `src/server.ts` | Express app setup and route registration |
| `src/handlers/githubHandler.ts` | Parse GitHub webhook events |
| `src/handlers/stripeHandler.ts` | Parse Stripe webhook events |
| `src/handlers/sentryHandler.ts` | Parse Sentry webhook events |
| `src/services/slackNotifier.ts` | Format and send Slack messages |
| `src/config.ts` | Channel mappings and credentials |
| `k8s/deployment.yaml` | Kubernetes deployment manifest |

## Implementation Steps

### Step 1: Server Setup

Three POST endpoints: `/webhooks/github`, `/webhooks/stripe`, `/webhooks/sentry`.
Standard Express JSON middleware.

### Step 2: Configuration

`src/config.ts`:
```typescript
export const config = {
  slack: {
    botToken: process.env.SLACK_BOT_TOKEN,
    channels: {
      github: process.env.SLACK_CHANNEL_GITHUB || '#dev-github',
      stripe: process.env.SLACK_CHANNEL_STRIPE || '#billing-alerts',
      sentry: process.env.SLACK_CHANNEL_SENTRY || '#dev-errors'
    }
  },
  apiKeys: {
    apiKey: process.env.API_KEY,
    webhookUrl: process.env.WEBHOOK_URL
  }
};
```

### Step 3: Provider Handlers

Each handler extracts relevant fields and returns a `SlackMessage`:
- **GitHub**: switch on event type (push, pull_request), format repo/author/action
- **Stripe**: handle `invoice.payment_failed`, format customer/amount
- **Sentry**: extract error title, project, severity level

### Step 4: Slack Notifier

Uses `@slack/web-api` `WebClient` to `chat.postMessage` with channel and text.

### Step 5: Kubernetes Deployment

2 replicas, port 3000, env vars from `webhook-secrets` K8s Secret
(SLACK_BOT_TOKEN, API_KEY, WEBHOOK_URL).
