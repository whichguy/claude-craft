# Plan: Migrate Authentication from Legacy to OAuth2

## Context

Our application uses a custom authentication system (`legacyAuth`) built in 2019. It
stores passwords with SHA-256 (no salt), sessions in a MySQL table, and has no support
for MFA or SSO. We need to migrate to a modern OAuth2/OIDC flow using Auth0 as the
identity provider.

## Current State

- Node.js 20, TypeScript 5.3, Express 4.18
- `src/auth/legacyAuth.ts` — current auth module:
  - `legacyAuth.login(email, password)` — SHA-256 hash comparison
  - `legacyAuth.validateSession(sessionId)` — MySQL session lookup
  - `legacyAuth.logout(sessionId)` — delete session row
  - `legacyAuth.createUser(email, password)` — insert with SHA-256 hash
- Session table: `sessions(id, user_id, created_at, expires_at)`
- 14 route files use `legacyAuth.validateSession()` as middleware
- 3 admin routes use `legacyAuth.createUser()` for user provisioning
- 45,000 active users with SHA-256 password hashes

## Approach

Build a new OAuth2 authentication module using Auth0, migrate all routes to use it,
and handle the user migration from local passwords to Auth0-managed identities.

## Files to Modify

- `src/auth/newAuth.ts` (new) — Auth0-based authentication module
- `src/auth/tokenValidator.ts` (new) — JWT token validation
- `src/auth/userMigration.ts` (new) — migrate users from legacy to Auth0
- `src/middleware/authMiddleware.ts` (new) — Express middleware for token validation
- `src/routes/auth.ts` (new) — login/logout/callback routes for OAuth2 flow
- `src/config/auth0Config.ts` (new) — Auth0 tenant configuration
- `src/types/auth.ts` (new) — authentication type definitions

## Implementation

### Phase 1: Auth0 Setup & Configuration

1. Create `auth0Config.ts`:
   - Auth0 domain, client ID, client secret from environment variables
   - Audience, callback URL, logout URL configuration
   - JWKS endpoint for token verification

2. Define auth types in `src/types/auth.ts`:
   ```typescript
   interface AuthUser {
     id: string;
     email: string;
     roles: string[];
     permissions: string[];
     auth0Id: string;
   }
   interface TokenPayload {
     sub: string;
     email: string;
     iss: string;
     aud: string;
     exp: number;
   }
   ```

### Phase 2: New Auth Module

1. Create `newAuth.ts`:
   - `newAuth.initiateLogin()` — redirect to Auth0 Universal Login
   - `newAuth.handleCallback(code)` — exchange code for tokens
   - `newAuth.validateToken(token)` — verify JWT with JWKS
   - `newAuth.logout(token)` — revoke token and redirect to Auth0 logout
   - `newAuth.getUserInfo(token)` — fetch user profile from Auth0

2. Create `tokenValidator.ts`:
   - Download JWKS from Auth0 on startup, cache with 1-hour TTL
   - Validate JWT signature, expiry, issuer, audience
   - Extract user claims from token payload

3. Create `authMiddleware.ts`:
   - Extract Bearer token from Authorization header
   - Validate token via `tokenValidator`
   - Attach user to `req.user`
   - Return 401 for missing/invalid tokens

### Phase 3: Route Migration

1. Create `src/routes/auth.ts`:
   - `GET /auth/login` — initiates Auth0 login flow
   - `GET /auth/callback` — handles Auth0 redirect with code
   - `POST /auth/logout` — revokes token, clears session
   - `GET /auth/me` — returns current user profile

2. Update all 14 route files:
   - Replace `legacyAuth.validateSession` middleware with `authMiddleware`
   - Update `req.session.userId` references to `req.user.id`
   - Update admin routes to check Auth0 roles instead of legacy role column

### Phase 4: User Migration

1. Create `userMigration.ts`:
   - Batch import users to Auth0 via Management API
   - Map legacy user IDs to Auth0 user IDs in a mapping table
   - Set `password_hash` in Auth0 import (they support SHA-256 import)
   - Users can log in with existing passwords initially
   - Auth0 will prompt for password reset on first login (re-hash with bcrypt)

2. Migration execution:
   - Export 45,000 users from local database
   - Import to Auth0 in batches of 500 (API rate limits)
   - Verify import: count match, spot-check 100 random users
   - Update local user table: add `auth0_id` column, populate from mapping

## Verification

1. Unit test token validator: valid JWT → pass, expired → reject, bad signature → reject
2. Unit test auth middleware: token present → attach user, missing → 401
3. Integration test: full OAuth2 flow (login → callback → token → protected route)
4. Integration test: logout flow (revoke → verify token is invalid)
5. Migration test: import 100 test users, verify login works with old passwords
6. Security test: verify old session-based auth cannot access new endpoints
