# Plan: Webhook Handler for Payment Provider

## Context

Our payment provider (Stripe) sends webhook events for payment status changes. We need
to receive these webhooks, validate their authenticity, and update our order database
accordingly. Events include: `payment_intent.succeeded`, `payment_intent.failed`,
`charge.refunded`, and `charge.disputed`.

## Current State

- Node.js 20, TypeScript 5.3, Express 4.18
- Stripe SDK installed but only used for creating payment intents
- No webhook handling infrastructure
- Orders table: `id`, `status`, `payment_intent_id`, `amount`, `customer_id`
- Order statuses: `pending`, `paid`, `failed`, `refunded`, `disputed`

## Approach

Add a webhook endpoint that receives Stripe events, validates the webhook signature,
and dispatches to event-specific handlers that update order status.

## Files to Modify

- `src/routes/webhooks.ts` (new) — webhook route definition
- `src/services/webhookService.ts` (new) — webhook processing logic
- `src/services/webhookHandlers.ts` (new) — per-event-type handlers
- `src/middleware/webhookAuth.ts` (new) — signature verification middleware
- `src/types/webhook.ts` (new) — webhook event types
- `src/app.ts` — register webhook route (raw body parser for signature verification)

## Implementation

### Phase 1: Webhook Endpoint & Auth

1. Add raw body middleware to `app.ts` for webhook route:
   - `app.use('/webhooks/stripe', express.raw({ type: 'application/json' }))`
   - Must be before JSON body parser to preserve raw body for signature verification

2. Create `webhookAuth.ts` middleware:
   - Extract `stripe-signature` header
   - Verify using `stripe.webhooks.constructEvent(rawBody, sig, endpointSecret)`
   - Reject invalid signatures with 401
   - Parse verified event payload

3. Create `webhooks.ts` route:
   - `POST /webhooks/stripe` — receives all Stripe events
   - Apply `webhookAuth` middleware
   - Pass verified event to `webhookService.processEvent()`
   - Return 200 immediately (process async to avoid Stripe timeout)

### Phase 2: Event Processing

1. Create `webhookService.ts`:
   - `processEvent(event: StripeEvent): Promise<void>`
   - Route event to appropriate handler based on `event.type`
   - Log all received events with event ID and type
   - Idempotency: check if event ID already processed (store in `webhook_events` table)
   - Return early if duplicate event

2. Create `webhookHandlers.ts` with per-event handlers:
   - `handlePaymentSucceeded(event)`:
     TODO: implement validation logic — verify payment amount matches order total,
     check for duplicate processing, update order status

   - `handlePaymentFailed(event)`:
     TODO: implement — update order status, notify customer

   - `handleChargeRefunded(event)`:
     TODO: implement — calculate refund amount, update order, adjust inventory

   - `handleChargeDisputed(event)`:
     TODO: implement — flag order, notify fraud team, gather evidence

### Phase 3: Database & Monitoring

1. Create `webhook_events` table for idempotency:
   - `event_id` (Stripe event ID), `type`, `processed_at`, `status`
   - Unique constraint on `event_id`

2. Add webhook metrics:
   - Counter: events received by type
   - Counter: events processed successfully vs failed
   - Histogram: processing time per event type

3. Add alerting for webhook failures:
   - >5% failure rate in 5-minute window → PagerDuty alert
   - No events received in 1 hour → warning (possible endpoint issue)
