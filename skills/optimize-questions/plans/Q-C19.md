# Plan: Webhook-Driven Order Processing System

## Context

We're integrating with Stripe webhooks to automate order creation. When a payment
succeeds, Stripe sends a `checkout.session.completed` event to our endpoint. We need
to create an order in our database and trigger fulfillment.

## Current Architecture

- Express.js API
- PostgreSQL database with `orders` table
- Manual order creation via admin panel
- No webhook integration

## Files to Change

| File | Change |
|------|--------|
| `src/routes/webhookRoutes.ts` | New POST `/webhooks/stripe` endpoint |
| `src/handlers/stripeWebhookHandler.ts` | Parse event, create order |
| `src/services/orderService.ts` | New `createOrder()` function |
| `src/services/fulfillmentService.ts` | Trigger shipping/email on new order |
| `src/models/order.ts` | Order model and DB schema |
| `migrations/003_create_orders.sql` | Orders table migration |

## Approach

Expose a webhook endpoint that Stripe calls on payment events. Verify the signature,
extract session data, create an order record, and kick off fulfillment.

## Implementation Steps

### Step 1: Orders Table Migration

```sql
CREATE TABLE orders (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  customer_email VARCHAR(255) NOT NULL,
  amount_cents INTEGER NOT NULL,
  currency VARCHAR(3) NOT NULL,
  status VARCHAR(50) DEFAULT 'pending',
  stripe_session_id VARCHAR(255),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_orders_email ON orders(customer_email);
```

### Step 2: Webhook Endpoint

`src/routes/webhookRoutes.ts`:
```typescript
router.post('/webhooks/stripe', express.raw({ type: 'application/json' }), stripeWebhookHandler);
```
- Use raw body parser for Stripe signature verification
- Mount at `/webhooks/stripe`

### Step 3: Webhook Handler

`src/handlers/stripeWebhookHandler.ts`:
```typescript
async function stripeWebhookHandler(req: Request, res: Response) {
  const sig = req.headers['stripe-signature'];
  const event = stripe.webhooks.constructEvent(req.body, sig, WEBHOOK_SECRET);

  if (event.type === 'checkout.session.completed') {
    const session = event.data.object;
    const order = await orderService.createOrder({
      customerEmail: session.customer_details.email,
      amountCents: session.amount_total,
      currency: session.currency,
      stripeSessionId: session.id,
    });
    await fulfillmentService.triggerFulfillment(order);
  }

  res.status(200).json({ received: true });
}
```

### Step 4: Order Service

`src/services/orderService.ts`:
- `createOrder(data)` — inserts a new row into `orders` table
- Returns the created order with generated UUID
- Sets initial status to `pending`

### Step 5: Fulfillment Service

`src/services/fulfillmentService.ts`:
- `triggerFulfillment(order)` — sends confirmation email and queues shipping
- Updates order status to `processing`
- Logs fulfillment initiation

### Step 6: Signature Verification

Stripe signature verification ensures requests are authentic:
- Uses `STRIPE_WEBHOOK_SECRET` environment variable
- Rejects requests with invalid signatures (returns 400)

## Verification

- [ ] Webhook endpoint accepts POST with valid Stripe signature
- [ ] Invalid signatures return 400
- [ ] `checkout.session.completed` creates an order in DB
- [ ] Order record has correct amount, email, session ID
- [ ] Fulfillment service is triggered after order creation
- [ ] Endpoint returns 200 to Stripe promptly

## Error Handling

- Signature verification failure: return 400, log warning
- Database insert failure: return 500, Stripe will retry
- Fulfillment failure: log error, order stays in `pending` (manual review)

## Rollback

- Remove webhook endpoint from Stripe dashboard
- Drop migration (no existing data depends on orders table)
- Revert route registration
