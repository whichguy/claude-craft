# Plan: Authentication System Overhaul

## Context

Our current auth system uses server-rendered session cookies with express-session backed by a PostgreSQL store. Sessions are causing scaling issues because every API request hits the session store. We need to migrate to a stateless token-based approach to eliminate server-side session storage and allow horizontal scaling without sticky sessions.

## Approach

Replace cookie-based sessions with stateless JWT tokens. JWTs are self-contained ‚Äî the server validates them by checking the signature, requiring zero server-side storage. This eliminates the session store bottleneck entirely and allows any API node to handle any request.

## Files to Modify

- `src/auth/jwt.ts` ‚Äî new JWT signing/verification module
- `src/auth/strategies.ts` ‚Äî replace session strategy with JWT strategy
- `src/middleware/authenticate.ts` ‚Äî new stateless auth middleware
- `src/routes/auth.ts` ‚Äî login/logout/refresh endpoints
- `src/models/User.ts` ‚Äî add refresh token methods
- `src/redis/session-store.ts` ‚Äî new Redis session management
- `config/auth.yml` ‚Äî JWT and session configuration
- `package.json` ‚Äî add `jsonwebtoken`, `ioredis`

## Implementation

### Phase 1: Stateless JWT Foundation

1. Install dependencies:
   ```
   npm install jsonwebtoken ioredis @types/jsonwebtoken
   ```

2. Create `config/auth.yml`:
   ```yaml
   jwt:
     algorithm: RS256
     access_token_ttl: 15m
     refresh_token_ttl: 7d
     issuer: myapp-api
   ```

3. Create `src/auth/jwt.ts`:
   - Load RSA key pair from `config/keys/` directory
   - `signAccessToken(payload)` ‚Äî short-lived, contains userId + roles
   - `signRefreshToken(payload)` ‚Äî long-lived, contains only userId
   - `verifyToken(token)` ‚Äî stateless signature verification only
   - No database or cache lookups during verification ‚Äî pure crypto check
   - This ensures zero server-side storage for token validation

4. Create `src/middleware/authenticate.ts`:
   - Extract Bearer token from Authorization header
   - Call `verifyToken()` ‚Äî if valid, attach decoded payload to `req.user`
   - Return 401 if expired or invalid signature
   - Entirely stateless ‚Äî no session store, no database hit

5. Update `src/routes/auth.ts`:
   - `POST /auth/login` ‚Äî validate credentials, return access + refresh tokens
   - `POST /auth/refresh` ‚Äî verify refresh token, issue new access token
   - `POST /auth/logout` ‚Äî return 200 (client discards tokens, nothing server-side)

### Phase 2: Role-Based Access Control

6. Create `src/auth/rbac.ts`:
   - Define role hierarchy: `viewer < editor < admin < superadmin`
   - `requireRole(minRole)` middleware factory
   - Roles embedded in JWT claims ‚Äî no database lookup needed

7. Update protected routes to use `requireRole()`:
   - Admin routes: `requireRole('admin')`
   - Editor routes: `requireRole('editor')`
   - Public routes: no middleware

8. Add role management endpoints:
   - `PUT /admin/users/:id/role` ‚Äî superadmin only
   - Validate role transitions (cannot self-escalate)

### Phase 3: Token Revocation and Session Management

9. Set up Redis connection in `src/redis/client.ts`:
   - Connection pool with retry logic
   - Health check endpoint integration

10. Create `src/redis/session-store.ts`:
    - Store active session data in Redis, keyed by userId
    - Track device info, IP address, and last activity per session
    - Set TTL matching the refresh token expiry (7 days)

11. Implement token revocation in `src/auth/revocation.ts`:
    - On logout: add the token's `jti` to a Redis blocklist
    - On password change: revoke all sessions for the user via Redis
    - On suspicious activity: admin can revoke all tokens for a user
    - `isRevoked(jti)` checks the Redis blocklist during token verification

12. Update `src/middleware/authenticate.ts`:
    - After signature verification, check `isRevoked(jti)` against Redis
    - If revoked, return 401 even though signature is valid

13. Add session management endpoints:
    - `GET /auth/sessions` ‚Äî list active sessions from Redis for current user
    - `DELETE /auth/sessions/:id` ‚Äî revoke a specific session
    - `DELETE /auth/sessions` ‚Äî revoke all sessions (logout everywhere)

### Phase 4: Migration and Cleanup

14. Create database migration to add `refresh_tokens` table:
    - Columns: `id`, `user_id`, `token_hash`, `device_info`, `expires_at`
    - Index on `user_id` for fast lookup

15. Remove old session dependencies:
    - Uninstall `express-session` and `connect-pg-simple`
    - Remove session middleware from `src/server.ts`
    - Delete `src/session/` directory

16. Update `src/models/User.ts`:
    - Remove `session`-related methods
    - Add `revokeAllTokens()` method that clears Redis entries
