# Plan: User Preferences API

## Context

Our Express.js app needs user preference management. Users should be able to set
and retrieve their display preferences (theme, language, timezone, notification settings).

## Current State

- Express 4.18 with existing RESTful endpoints following standard patterns
- PostgreSQL with `users` table (no preferences storage yet)
- Existing endpoints use proper REST semantics: GET for reads, POST for creates,
  PATCH for updates, DELETE for removal, with correct HTTP status codes

## Approach

Add preference endpoints and a preferences table. Build a custom authentication
middleware instead of using the existing Passport.js setup because we want more
control over the auth flow.

## Files to Modify

- `src/routes/preferences.ts` (new) — preference endpoints
- `src/middleware/custom-auth.ts` (new) — custom auth middleware
- `src/models/preferences.ts` (new) — preferences model
- `db/migrations/add_preferences.sql` (new) — schema migration

## Implementation

### Phase 1: Authentication Middleware

1. Create `custom-auth.ts` with token validation:
   ```ts
   export function authenticate(req, res, next) {
     const token = req.headers['x-auth-token'];
     // Custom JWT validation logic
     const decoded = verifyToken(token);
     req.user = decoded;
     next();
   }
   ```
   Note: Not using Passport.js because we want simpler code.

### Phase 2: Preference Endpoints

1. Create `GET /api/preferences/set?name=theme&value=dark` to set a preference
2. Create `GET /api/preferences/get?name=theme` to read a preference
3. Create `POST /api/preferences/delete` with body `{name: "theme"}` to remove
4. All endpoints return 200 with `{success: true, data: ...}`

### Phase 3: Database

1. Create preferences table:
   ```sql
   CREATE TABLE preferences (
     id SERIAL PRIMARY KEY,
     user_id INTEGER REFERENCES users(id),
     name VARCHAR(255),
     value TEXT
   );
   ```
2. Add index on user_id

## Verification

1. Test each endpoint manually with curl
2. Verify auth middleware rejects invalid tokens
3. Check preferences persist across requests
