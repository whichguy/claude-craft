# validate-questions Skill

End-to-end validation pipeline for review-plan questions. Determines whether each
question creates material value when applied to realistic implementation plans.

**Architecture:** Four-stage pipeline — ablation, overlap investigation, analytical
derivation, and synthesis. Each stage produces intermediate artifacts; the final stage
synthesizes into a Question Effectiveness Report with actionable dispositions.

**Bench bank:** `skills/review-plan/inputs/bench/*.md` — realistic plans generated by
Claude Code plan mode, covering diverse domains and quality levels. The bench bank must
have sufficient variance (rough/medium/good plans) for meaningful signal.

**Relationship to question-bench:** This skill is the orchestrator; question-bench is the
engine. question-bench runs a single ablation experiment. This skill runs the full
validation pipeline: ablation + overlap + analysis + report.

## Stage 1 — Full Ablation (E=3)

**Skip if:** SKIP_ABLATION or REPORT_ONLY

**Purpose:** Measure whether gate:1, gate:1+2, and all questions produce progressively
better plan improvements. Identifies per-question hit rates, per-dimension impacts,
overlap candidates, never-fired questions, and the minimal effective set.

**Execute:**
```
/question-bench BENCH_DIR --experiments 3 --questions all
```

This invokes the question-bench skill which handles batching (4 plans per batch),
evaluation, editing, scope gating, judging, and reporting.

**Capture output:** The question-bench report contains 8 sections. Extract and store:

| Artifact | Source Section | Stored As |
|----------|---------------|-----------|
| Ablation summary | Section 1 | ABLATION_SUMMARY |
| Per-dimension impact | Section 2 | DIMENSION_IMPACT |
| Per-question hit rates | Section 3 | QUESTION_HITS |
| Overlap candidates | Section 4 | OVERLAP_CANDIDATES |
| Equivalence analysis | Section 5 | EQUIVALENCE |
| Never-fired questions | Section 6 | NEVER_FIRED |
| Quality verdict | Section 7 | QUALITY_VERDICT |
| Preliminary dispositions | Section 8 | PRELIMINARY_DISPOSITIONS |

**Validation:** Confirm the ablation produced meaningful signal:
- At least one experiment has verdict IMPROVED (spread > 0.15)
- Hit rates span a range (not all 0% or all 100%)
- If all experiments are NEUTRAL: warn "bench bank may lack sufficient defects"

## Stage 2 — Overlap Investigation

**Skip if:** SKIP_OVERLAP or REPORT_ONLY

**Purpose:** For question pairs flagged as overlapping in Stage 1, determine whether
they are truly redundant (one subsumes the other) or complementary (both contribute
unique value).

**Identify suspect pairs from Stage 1:**
```
pairs = []
For each overlap candidate from OVERLAP_CANDIDATES:
    # Find a plan where both questions fire (from QUESTION_HITS)
    test_plan = find_plan_where_both_fire(q_a, q_b, QUESTION_HITS)
    IF test_plan found:
        pairs.append({q_a, q_b, test_plan})
```

**For each suspect pair, run targeted ablation:**

For each pair (Q-A, Q-B) on test_plan:

1. **Generate 3 planBs** (spawn 3 editor agents in parallel):
   - planB-A: Apply only Q-A's finding to the original
   - planB-B: Apply only Q-B's finding to the original
   - planB-AB: Apply both Q-A and Q-B findings to the original

   Use the same editor prompt pattern as question-bench Step 2b, but with
   single-question findings from Stage 1's EVAL_VERDICTS.

2. **Judge 3 comparisons** (spawn 3 judge agents in parallel):
   - Original vs planB-A
   - Original vs planB-B
   - Original vs planB-AB

   Use position-randomized question-bench-judge agents.

3. **Compute spreads** using question-bench scoring:
   ```
   strength_weight = {"strong": 1.0, "moderate": 0.67, "slight": 0.33}
   MAX_SCORE = 8.0

   For each comparison:
       score_planB = sum(strength_weight[q.strength] for q in results if q.winner == "planB")
       score_orig = sum(strength_weight[q.strength] for q in results if q.winner == "original")
       spread = (score_planB - score_orig) / MAX_SCORE
   ```

4. **Apply overlap interpretation rules:**
   ```
   spread_A = spread of planB-A
   spread_B = spread of planB-B
   spread_AB = spread of planB-AB

   IF spread_AB > max(spread_A, spread_B) + 0.05:
       verdict = "COMPLEMENTARY" — both KEEP
   ELIF abs(spread_AB - spread_A) <= 0.05:
       verdict = "B_SUBSUMED" — Q-B subsumed by Q-A, MERGE candidate
   ELIF abs(spread_AB - spread_B) <= 0.05:
       verdict = "A_SUBSUMED" — Q-A subsumed by Q-B, MERGE candidate
   ELIF spread_AB < max(spread_A, spread_B):
       verdict = "HARMFUL_COMBO" — one question hurts when combined, investigate
   ELSE:
       verdict = "INCONCLUSIVE" — try another test plan
   ```

**State output:** OVERLAP_RESULTS — array of {q_a, q_b, test_plan, spread_A, spread_B,
spread_AB, verdict, per_dimension_breakdown}

## Stage 3 — Analytical Derivation

**Skip if:** REPORT_ONLY

**Purpose:** Derive cluster contributions and layer contributions from Stage 1 data
without running additional ablation experiments. This is cheaper than full experimental
runs and provides sufficient directional signal.

### Cluster contribution analysis

From QUESTION_HITS, group questions by their L2 cluster. For each cluster:
```
For each cluster C:
    fired_questions = [q for q in C.questions if QUESTION_HITS[q].hit_rate > 0]
    total_questions = len(C.questions)
    cluster_hit_range = (min_hit_rate, max_hit_rate) of fired_questions
    
    # Attribute dimension impacts from Stage 1's per-question impact data
    primary_dimensions = most-improved Q-PQ dimensions attributed to C's questions
    
    cluster_rank = sort by: len(fired_questions), then max hit_rate
```

### Layer contribution analysis

From ABLATION_SUMMARY:
```
L1_contribution = spread(gate:1+2) — spread attributable to L1 questions
L2_epilogue_contribution = spread(all) - spread(gate:1+2) — attributable to L2 + epilogue
incremental_L2 = L2_epilogue_contribution

Both layers contribute independently if each increment > 0.05
```

**State output:** CLUSTER_ANALYSIS, LAYER_ANALYSIS

## Stage 4 — Synthesis & Report

**Purpose:** Produce the final Question Effectiveness Report by synthesizing all
stage outputs into per-question dispositions with evidence.

### Disposition algorithm

For each question Q-ID in the full question set:

```
IF Q-ID in NEVER_FIRED:
    disposition = "DROP"
    evidence = "0% hit rate across N plans"
    confidence = "HIGH"

ELIF Q-ID in OVERLAP_RESULTS where verdict == "B_SUBSUMED" or "A_SUBSUMED":
    disposition = "MERGE"
    merge_target = the non-subsumed question
    evidence = "Run 3b: spread(combined) ≈ spread(target), delta = X"
    confidence = "HIGH"

ELIF Q-ID in OVERLAP_RESULTS where verdict == "HARMFUL_COMBO":
    disposition = "MERGE"
    evidence = "Run 3b: combined spread < individual spread, dedup needed"
    confidence = "HIGH"

ELIF QUESTION_HITS[Q-ID].hit_rate < 0.15 AND fires only on specific domains:
    disposition = "CONDITIONAL"
    condition = inferred domain flag (HAS_UNTRUSTED_INPUT, HAS_STATE, etc.)
    evidence = "fires on N/M plans, all in domain X"
    confidence = "MEDIUM"

ELIF QUESTION_HITS[Q-ID].hit_rate >= 0.25:
    disposition = "KEEP"
    evidence = "hit rate N%, improves dimensions: X, Y"
    confidence = "HIGH" if hit_rate >= 0.5 else "MEDIUM"

ELSE:
    disposition = "KEEP"
    evidence = "insufficient data for demotion"
    confidence = "LOW"
```

## Maintenance: Updating the Bench Bank

The bench bank should be refreshed when:

1. **New question clusters are added** — ensure bench plans trigger the new cluster's conditions
2. **Disposition changes are applied** — re-run to verify the changes improve efficiency without quality loss
3. **Claude Code's plan style changes** — regenerate plans to match current output patterns

To add a new bench plan:
```
1. Identify the gap (which cluster/condition lacks triggering plans)
2. Write a natural feature request for that domain
3. Have Claude Code generate the plan in plan mode
4. Save to skills/review-plan/inputs/bench/bench-<name>.md
5. Re-run /validate-questions to incorporate
```
